# lazy load

通过基本的webpack配置，我们可以轻松实现按需打包，最终输出一个js文件.

但是，在真正的生产环境下，如果只输出一个过大的js文件，会严重影响页面的加载速度，为了解决这个问题，我们需要做以下几点改进：
1. 不将第三方库文件打包进bundle.
2. css抽离成独立的文件.
3. 基本的 lazy load.
4. 基于框架的lazy load



## 组件的 lazy load (异步加载)
现在主流的前端框架基本都倡导UI组件化，
组件lazy load 的实现实际上并不是webpack提供的功能，它是由各个框架自己提供的方法实现的.
