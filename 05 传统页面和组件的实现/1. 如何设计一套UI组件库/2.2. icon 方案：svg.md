# svg ICON 方案
https://www.keycdn.com/blog/icon-fonts-vs-svgs
https://css-tricks.com/using-svg/

## 1. 最简单的svg用法：直接当作image来使用。（适用简单场景）
1. <image src="xxx.svg">
2. css中background-image：url('xxx.svg')
在这里实际上，只是简单地把svg元素当做了image元素来处理了, 这种用法并不能充分发挥SVG的特性，比如可以通过css自由改变icon的颜色.   
实际上，我们可能更需要的是: <svg src="xxx.svg">, 但实际上这样方式是被web禁止的. 原因如下： 因为svg是以xml的形式展现的，如果通过src的方式直接插入到DOM中，会有严重的安全隐患. 例如，恶意的svg文件可能会在内容中插入<script/>脚本，从而引发xss攻击.  
如果想使用lazy load 的方式来加载SVG，需要做两件事情：1. 配置一个SVG资源的服务，前端手动请求资源，2. web部分需要加载完svg的内容后（并做好安全检测），然后自己插入到DOM中。这样做的成本比较高。  

## 2. inline svg 用法 （高级用法）
ant design 的 ICON 方案采用的是内联svg实现，所有的SVG都是预打包在代码里。
antd的方法：
1. svg 以结构化的方式存储在代码中, xml --> json：@ant-design/icons
2. 将svg数据对象(json对象)封装为React组件，json --> react component: @ant-design/icons-react
3. 在 antd 中定义一个ICON组件，预加载所有的SVG对象。如果其他组件中对 ICON 有引用，则其他组件引用ICON组件即可。通过webpack的特性进行按需加载，不重复打包资源。
4. 注意：antd 中定义了三种风格的SVG图标：fill, outline, twotone.

以上代码中用到的函数定义：
```js
var normalViewBox = '0 0 1024 1024';
var newViewBox = '64 64 896 896';
var fill = 'fill';
var outline = 'outline';
var twotone = 'twotone';
function getNode(viewBox) {
    var paths = [];
    for (var _i = 1; _i < arguments.length; _i++) {
        paths[_i - 1] = arguments[_i];
    }
    return {
        tag: 'svg',
        attrs: { viewBox: viewBox },
        children: paths.map(function (path) {
            if (Array.isArray(path)) {
                return {
                    tag: 'path',
                    attrs: {
                        fill: path[0],
                        d: path[1]
                    }
                };
            }
            return {
                tag: 'path',
                attrs: {
                    d: path
                }
            };
        })
    };
}
function getIcon(name, theme, icon) {
    return {
        name: name,
        theme: theme,
        icon: icon
    };
}
```
说明：
1. 从以上代码中可以很容易看出来一个SVG对象的结构，需要注意的是对于twotone图标的icon属性是一个函数，这样做的目的是为了延迟赋值。
2. svg的JSON对象并不手动写的，而是自己编写nodejs工具，根据svg文件生成了这些代码。

从JSON对象封装成React对象，ReactIcon：
```js
class Icon extends React.Component<IconProps> {
  render() {
    ...
    return generate(target.icon as AbstractNode, `svg-${target.name}`, {
      className,
      onClick,
      style,
      ['data-icon']: target.name,
      width: '1em',
      height: '1em',
      fill: 'currentColor',
      ['aria-hidden']: 'true',
      ...rest
    });
  }
}

export function generate(
  node: AbstractNode,
  key: string,
  rootProps?: { [key: string]: any } | false
): any {
  if (!rootProps) {
    return React.createElement(
      node.tag,
      { key, ...normalizeAttrs(node.attrs) },
      (node.children || []).map((child, index) =>
        generate(child, `${key}-${node.tag}-${index}`)
      )
    );
  }
  return React.createElement(
    node.tag,
    {
      key,
      ...normalizeAttrs(node.attrs),
      ...rootProps
    },
    (node.children || []).map((child, index) =>
      generate(child, `${key}-${node.tag}-${index}`)
    )
  );
}
```

antd 中的ICON对象的作用：
1. 预加载所有的svg对象资源，
2. 预设一个primary color，它是ReactIcon中存储的一个全局变量，所有的ICON都是一种主题色。可以后面动态修改，但是需要手动触发所有的ICON的对象的render。  
3. 在svg的最外层加了一个<i>元素。
